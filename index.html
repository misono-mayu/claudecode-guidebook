How it works

スキルの仕組みと拡張性

スライドが作られるまでの流れと、自分好みにカスタマイズできるポイントを紹介します。スキルは誰でも編集・拡張できる設計になっています。

P0
PHASE 0 — Context Loading
デザイン・ルールの読み込み

スキルが起動するとまず共有ファイルを読み込みます。カラー・フォント・余白などのデザイン定義と、出力フォーマットのロジックが読み込まれ、以降の全フェーズに引き継がれます。

カスタマイズポイント
~/.claude/skills/shared/slide-design.md を編集するだけで全スキルのデザインベースが一括変更されます。自社ブランドカラーへの変更やフォントの差し替えもここで行います。
P1
PHASE 1 — ヒアリング・構成精査
種別の選択とスライド骨格の決定

「リアルイベント / ウェビナー / 企画書スライド化」を選択し、それぞれに合ったヒアリングを進めます。伝えたいメッセージを1つに絞り、タイムテーブルとスライド構成の骨格を固めます。

カスタマイズポイント
SKILL.md の Phase 1 を編集して、新しい種別（例：「社内研修」「採用説明会」）を追加できます。種別ごとに異なるヒアリング項目や構成フォーマットを定義するだけで、AIが自動で使い分けます。
P2
PHASE 2 — スライド詳細展開 + 自動レビュー
各スライドの中身を記述し、自動レビューで修正

骨格をもとに各スライドに「実際に表示するテキスト」と「話す内容」を記述します。1スライド1メッセージの原則を守り「見せる」と「話す」の役割を分離します。完成後はAIが自動でレビューし、問題点を修正してから次へ進みます。

カスタマイズポイント
Phase 2 のルールに独自の制約を追記できます。さらに、自分の教育思想や伝え方を書いた teaching_style.md を Phase 0 で読み込むよう設定すれば、AIがそのスタイルを反映してスライドを展開します。
P3
PHASE 3 — 出力フォーマット選択
HTML / PowerPoint / NotebookLM を選択して生成

コンテンツが完成したら出力形式を選びます。3形式すべてで同じコンテンツが使われるため、あとから形式を変えることも可能です。PowerPointの場合は参照テンプレート（slide_template.pptx）のデザインが自動で反映されます。

カスタマイズポイント
~/.claude/skills/shared/output-formats.md を編集することで出力ロジックを変更できます。slide_template.pptx を差し替えれば PowerPoint のデザインを丸ごと変えられます。
P4
PHASE 4 — 確認事項の提示
本番前のチェックリストを自動生成

生成後、種別に応じた確認事項リストが自動で出力されます。「時間配分に矛盾はないか」「資料・プロンプトは準備できているか」など、当日に向けた抜け漏れを防ぐチェックが入ります。

カスタマイズポイント
Phase 4 のチェック項目に、自分がよくやりがちなミスやチーム固有の確認事項を追記できます。過去のイベントで失敗したことを書き加えておけば、次回から自動でリマインドされます。

自分だけのスキルに育てる

Phase 0 で読み込むファイルを追加するだけで、AIがあなたの知識・スタイル・過去資産を反映したスライドを作るようになります。

過去の企画・構成

うまくいった回の構成を past_events.md にまとめておくと、AIが「前回のどこが良かったか」を反映して次の構成を提案します。

話し方・教育思想

「参加者が自走できる設計を重視する」などの哲学を teaching_style.md に書けば、話者ノートや構成に自分らしさが出ます。

専門知識・用語集

扱うテーマの知識や用語集を domain_knowledge.md に書いておくと、初心者向けの言い換えや補足スライドが的確になります。

参加者の声・FAQ

アンケート結果やよく来る質問を faq.md にまとめると、「つまずきやすいポイント」への言及が構成に自動で入ります。

Advanced

NotebookLM連携の詳しい使い方

ハンズオンでは触れなかった操作の詳細です。自分で試すときの参考にしてください。

1

前提確認

uv・notebooklm CLIのインストールとログインが完了していること（下の「はじめてのセットアップ」を参照）。

2

スキルを呼び出す

Claude Codeで /notebooklm-auto と入力するか、「NotebookLMでスライド作って」と話しかけることで自動起動します。

トリガーフレーズ
NotebookLMでスライド作って
3

ノートブックを選択する

既存のノートブックを使うか、新規作成するかを選びます。新規の場合はノートブック名を指示するとClaude Codeが作成します。

4

スライドが生成される

CUSTOMIZE.mdに定義されたデザインルール（カラー・フォント・ビジュアルパターン）が反映されたスライドがNotebookLMで生成されます。生成には数分かかります。

5

PDFをダウンロードする

生成完了後、カレントディレクトリにPDFとして保存されます。ファイル名は slides_YYYYMMDD.pdf 形式です。

6

デザインをカスタマイズする（応用）

~/.claude/skills/notebooklm-auto/CUSTOMIZE.md を編集することで、ビジュアルパターンや色の使い方を変更できます。

1

uv をインストールする

Python パッケージ管理ツールです。まず uv が入っているか確認します。

確認コマンド
which uv

「command not found」と表示された場合は、以下でインストールしてください。

インストール（Mac）
brew install uv
2

notebooklm CLI をインストールする

コマンドラインからNotebookLMを操作するためのツールです。

インストール
uv tool install "notebooklm-py[browser]"
確認コマンド
which notebooklm
3

NotebookLM にログインする

ブラウザが起動し、Googleアカウントでの認証が求められます。お使いのGoogleアカウントでログインしてください。

ログイン
notebooklm login
check_circle

ログイン後、notebooklm list を実行してエラーが出なければ準備完了です。
